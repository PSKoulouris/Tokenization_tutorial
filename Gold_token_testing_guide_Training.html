Luxembourg Gold Token

Smart Contract Testing Guide

Version: 1.0 | Date:

Overview

This comprehensive guide provides step-by-step instructions for testing the Luxembourg Gold Token smart contract. The contract implements an ERC-20 token backed by physical gold reserves, maintaining a 1:1 backing ratio.

Contract Features:
  • ERC-20 compliant gold-backed token
  • Owner-controlled minting and burning
  • Transparent gold reserve tracking
  • Real-time backing ratio calculation

Step 1: Environment Setup

Required Tools

  1. Remix IDE: Visit remix.ethereum.org
  2. MetaMask Wallet: Install browser extension
  3. Test ETH: From Sepolia faucet
  4. Contract Code: Luxembourg Gold Token source code

MetaMask Configuration

  1. Connect MetaMask to Sepolia test network
  2. Get test ETH from sepolia.dev or faucets.chain.link
  3. Ensure you have at least 0.1 ETH for gas fees

Remix IDE Setup

  1. Create new file: LuxembourgGoldToken.sol
  2. Paste the smart contract code
  3. Select Solidity compiler version 0.8.19
  4. Set environment to "Injected Provider - MetaMask"

Step 2: Contract Deployment

Deployment Process

  1. Compile Contract: Click "Compile LuxembourgGoldToken.sol"
  2. Check for Errors: Ensure no compilation errors
  3. Deploy Contract: Go to "Deploy & Run Transactions" tab
  4. Select Contract: Choose "LuxembourgGoldToken"
  5. Deploy: Click "Deploy" button
  6. Confirm Transaction: Approve in MetaMask
Important: Save the deployed contract address for reference. You'll need it for block explorer verification and future interactions.

Step 3: Initial State Verification

Test Initial Values: 1. Call name() → Should return "Luxembourg Gold Token" 2. Call symbol() → Should return "LUXGOLD" 3. Call totalSupply() → Should return 0 4. Call totalGoldReserves() → Should return 0 5. Call decimals() → Should return 18 6. Call owner() → Should return your address
Expected Results: All initial values should be correct, with zero supply and reserves until the first minting operation.

Step 4: Minting Process Testing

Basic Minting Test

  1. Call mint function: mint(YOUR_ADDRESS, 1000)
  2. Confirm transaction in MetaMask
  3. Wait for confirmation on blockchain

Verify Minting Results

Post-Mint Verification: 1. Call totalSupply() → Should show 1000 2. Call totalGoldReserves() → Should show 1000 3. Call balanceOf(YOUR_ADDRESS) → Should show 1000 4. Call getGoldPerToken() → Should show 1000000000000000000 (This represents 1.0 with 18 decimal places)

Multiple Minting Test

  1. Mint additional tokens: mint(YOUR_ADDRESS, 500)
  2. Verify total supply increases to 1500
  3. Verify gold reserves increase to 1500
  4. Confirm backing ratio remains 1.0

Step 5: Burning Process Testing

Basic Burning Test

  1. Call burn function: burn(500)
  2. Confirm transaction in MetaMask
  3. Wait for confirmation

Verify Burning Results

Post-Burn Verification: 1. Call totalSupply() → Should show 1000 2. Call totalGoldReserves() → Should show 1000 3. Call balanceOf(YOUR_ADDRESS) → Should show 1000 4. Call getGoldPerToken() → Should still show 1000000000000000000 (Ratio should remain 1.0)

Step 6: Access Control Testing

Owner Permission Testing

  1. Note current owner: Record the deploying address
  2. Switch MetaMask account to different address
  3. Try to mint: Call mint(address, amount)
  4. Expect failure: Should revert with "Ownable: caller is not the owner"
  5. Switch back: Return to owner account
  6. Verify mint works: Minting should succeed as owner
Security Check: This test confirms that only the contract owner can mint new tokens, preventing unauthorized token creation.
