Gold Token Backing Ratio

Understanding how digital tokens are backed by physical gold reserves

What is the Backing Ratio?

Core Concept

The gold backing ratio determines how much physical gold backs each digital token. In the Luxembourg Gold Token contract, this ratio should ideally be 1:1, meaning:

  • 1 token = 1 unit of gold (gram, ounce, etc.)
  • If you own 100 tokens, there should be 100 grams of gold in secure storage
  • This creates trust and ensures real value behind digital assets

The Smart Contract Formula

function getGoldPerToken() public view returns (uint256)
{
    if (totalSupply() == 0) return 0;
    return (totalGoldReserves * 10**decimals()) / totalSupply();
}

Why the Complex Math?

The multiplication by 10^18 is needed because:

  • Solidity doesn't handle decimal numbers directly
  • ERC-20 tokens use 18 decimal places for precision
  • 1.0 is represented as 1,000,000,000,000,000,000
  • This allows precise calculations without losing fractional amounts

How the Ratio is Maintained

When Minting (Adding Gold)

_mint(to, amount); // Increase token supply
totalGoldReserves += amount; // Increase gold reserves by same amount

Process: Physical gold is deposited ‚Üí Verified by auditors ‚Üí Corresponding tokens minted

When Burning (Removing Gold)

_burn(msg.sender, amount); // Decrease token supply
totalGoldReserves -= amount; // Decrease gold reserves by same amount

Process: Token holder burns tokens ‚Üí Physical gold withdrawn from storage ‚Üí Delivered to holder

Trust and Transparency

How Users Can Verify Backing

Anyone can check the backing ratio at any time by calling these functions:

  • totalSupply() - See total tokens in circulation
  • totalGoldReserves - See recorded gold reserves
  • getGoldPerToken() - Calculate the current ratio

Real-World Requirements

Maintaining perfect 1:1 backing in practice requires:

  • Secure storage of physical gold in audited vaults
  • Regular audits by independent third parties
  • Insurance against theft, loss, or damage
  • Transparent reporting of all reserves and transactions
  • Redemption processes for converting tokens back to physical gold
